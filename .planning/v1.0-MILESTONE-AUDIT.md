### Integration Issues

**1. Test harness break (Low severity, test-only)**

`tests/test-dyck.html` lines 59, 111-117 still import and assert on `THEME_COLORS` from `js/core/colors.js`, which was removed by Phase 8 Plan 02. Running the test harness will show 2 of 26 assertions failing. The production app is completely unaffected.

**2. Layout property name inconsistency (Maintenance hazard)**

`js/structures/binary-tree.js` uses `node.layoutX`/`node.layoutY` while bijection modules use `node._layoutX`/`node._layoutY`. Not a bug today (each path creates fresh trees via `fromDyck()`), but a forward hazard if future code shares tree objects between these paths without calling `fromDyck()` again.
