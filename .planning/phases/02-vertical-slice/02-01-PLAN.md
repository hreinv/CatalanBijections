Left subtree is the word between first +1 and its matching -1. Right subtree is everything after. Return `{ left, right }` node objects, null for leaves.
- `toDyck(node)`: Recursive: null returns []. Otherwise return `[1, ...toDyck(node.left), -1, ...toDyck(node.right)]`.
- `draw(ctx, instance, opts)`: Render binary tree within bounding box using simplified in-order layout (see 02-RESEARCH.md).

  - Handle null instance (empty tree for n=0 edge case) by drawing nothing.

**triangulation.js** (STRC-10):
- `fromDyck(dyckWord)`: Convert via binary tree intermediate (see 02-RESEARCH.md code example).
  1. Build binary tree from Dyck word (reuse the same 1L0R logic, implement locally or import from binary-tree.js — implement locally to keep modules independent).
  2. `treeToTriangulation(node, lo, hi, diagonals)`: Each internal node creates a triangle. Apex vertex = lo + countNodes(node.left) + 1. Add diagonal (lo, apex) if apex-lo > 1, add diagonal (apex, hi) if hi-apex > 1. Recurse left on [lo, apex], right on [apex, hi].
  3. Return `{ n, diagonals, numVertices: n+2 }`.
- `toDyck(instance)`: Convert triangulation back to Dyck word by reversing the process: reconstruct binary tree from diagonals, then serialize to Dyck word. Implementation: use `triangulationToTree(diagonals, lo, hi, numVertices)` that finds the apex splitting edge (lo, hi) and recurses. Then apply the standard tree-to-Dyck serialization.
- `draw(ctx, instance, opts)`: Draw (n+2)-gon with diagonals (see 02-RESEARCH.md).

  - Draw polygon edges with thick stroke.
  - Draw diagonals with slightly thinner stroke or different dash pattern to distinguish from edges.
  - Draw vertex labels (0..n+1) outside the circle.
  - Use `opts.colors` to shade triangles (fill each triangle region with a semi-transparent color).
