  <files>

  <action>
Create `js/engine/` directory and implement the animation engine as an ES module following Pattern 4 from 02-RESEARCH.md.

Export a factory function `createAnimationEngine(options)` that returns an engine object. The options parameter takes `{ onRender, getState, setState }` callbacks so the engine stays decoupled from the app controller.

**Engine internal state:**
- `animFrameId` (null or rAF ID for clean cancel)
- `lastTimestamp` (null or DOMHighResTimeStamp for delta computation)

**Exported engine API object:**
```javascript
{
  play(),        // Start rAF loop. Sets playing=true, resets lastTimestamp, schedules tick.
  pause(),       // Stop rAF loop. Sets playing=false, calls cancelAnimationFrame.
  togglePlay(),  // Convenience: if playing, pause; else play.
  stepForward(), // If steps exist: increment currentStep (clamp at steps.length-1), set progress to 0.0, pause if playing, trigger render.
  stepBackward(),// If steps exist: decrement currentStep (clamp at 0), set progress to 0.0, pause if playing, trigger render.
  jumpToStart(), // Set currentStep=0, progress=0.0, pause if playing, trigger render.
  jumpToEnd(),   // Set currentStep=steps.length-1, progress=1.0, pause if playing, trigger render.
  setSpeed(v),   // Clamp v to [0.5, 3.0], update animation.speed, no render needed.
  isPlaying(),   // Returns current playing state.
}
```

**tick(timestamp) function** (internal, not exported):
- Compute deltaMs from (timestamp - lastTimestamp) * speed.
- Update lastTimestamp.
- If steps array is empty, pause immediately and return (no-op animation).
- Advance progress: `progress += deltaMs / stepDuration` where stepDuration is a constant (e.g., 800ms per step — this can be tuned later).
- If progress >= 1.0:
  - If currentStep < steps.length - 1: advance to next step, set progress to 0.0 (carry over excess if desired, but simpler to reset).
  - Else: animation complete, call pause(), set progress to 1.0.
- Call `onRender()` to trigger canvas redraw.
- If still playing: schedule next tick via `requestAnimationFrame(tick)`.

**State access pattern:** The engine reads/writes animation state via `getState()` and `setState(changes)` callbacks, NOT by directly mutating a shared object. This keeps the engine module independent of the app controller's state structure. Example:
```javascript
// getState() returns { steps, currentStep, progress, playing, speed }
// setState({ currentStep: 2, progress: 0.0 }) merges into animation state
```

**Edge case handling:**
- play() when steps array is empty: immediately pause, no error.
- stepForward/stepBackward when steps array is empty: no-op, no error.
- play() when already playing: no-op (don't start duplicate loop).
- pause() when already paused: no-op.
- Speed clamped to [0.5, 3.0] with Math.max/Math.min.

**Anti-pattern avoidance:**
- Never use setInterval or frame counting. Always use deltaTime from rAF timestamp.
- Always cancelAnimationFrame before starting a new loop (prevent orphan loops per Pitfall 4).
- Always null out animFrameId after cancel.

Update main.js (from Plan 02-02) to import the animation engine and wire all playback controls.

**New import:**
```javascript
import { createAnimationEngine } from './engine/animation.js';
```

**Engine initialization** (inside DOMContentLoaded, after state setup):
- Create engine instance:
```javascript
const engine = createAnimationEngine({
  onRender: () => render(),
  getState: () => state.animation,
  setState: (changes) => {
    Object.assign(state.animation, changes);
    updatePlayPauseButton();
  },
});
```

**updatePlayPauseButton()**: Helper that sets the play/pause button text to "Pause" when playing, "Play" when paused. Also toggles any visual indicator.

**Wire playback button event handlers** (replace the `// Playback controls wired in Plan 02-03` comment):
- `#btn-play-pause` click: `engine.togglePlay()`
- `#btn-step-fwd` click: `engine.stepForward()`
- `#btn-step-back` click: `engine.stepBackward()`
- `#btn-start` click: `engine.jumpToStart()`
- `#btn-end` click: `engine.jumpToEnd()`

**Wire speed slider:**
- `#speed-slider` input event: Read value, call `engine.setSpeed(parseFloat(value))`, update `#speed-display` text to `${value}x`.

**Enable playback controls:**
- Remove any `disabled` class or attribute from playback buttons and speed slider that was set in Plan 02-02.
- Alternatively, if Plan 02-02 used CSS `.disabled` class, remove it from the playback control group.

**State change integration:**
- When structure selection or n changes (existing handlers from Plan 02-02), also call `engine.pause()` to stop any running animation and reset animation state:
  ```javascript
  engine.pause();

  state.animation.currentStep = 0;
  state.animation.progress = 0.0;
  ```
  This prevents orphan animation loops when changing structures (Pitfall 4 from research).

- Modify the render function's drawing behavior.
