  <action>
Completely rewrite main.js from the verification-pattern placeholder into the full app controller. Follow the App State Object pattern from 02-RESEARCH.md.

**Imports:**
- `setupCanvas`, `clearCanvas` from `./core/canvas-utils.js`
- `enumerate` from `./core/dyck.js`
- `CORRESPONDENCE_COLORS` from `./core/colors.js`
- `structures` from `./structures/registry.js`

**State object** (Pattern 2 from research):
```javascript
const state = {
  n: 3,
  instanceIndex: 0,
  sourceKey: 'parentheses',
  targetKey: 'dyck-path',
  dyckWords: [],
  currentDyck: null,
  animation: {
    steps: [],
    currentStep: 0,
    progress: 0.0,
    playing: false,
    speed: 1.0,
  }
};
```

**readTheme()**: Keep the existing readTheme function that reads CSS custom properties and returns a frozen theme object.

**populateDropdowns()**: Iterate `Object.entries(structures)` and add `<option value="key">label</option>` to both `#source-select` and `#target-select`. Set initial selected values to match `state.sourceKey` and `state.targetKey`.

**updateDerivedState()**: Called whenever n or instanceIndex changes. Sets `state.dyckWords = enumerate(state.n)`, clamps `state.instanceIndex` to valid range, sets `state.currentDyck = state.dyckWords[state.instanceIndex]`. Updates the `#instance-indicator` text to show `"X of Y"` format (1-indexed for display).

**render()**: The core rendering function (Pattern 3 from research).
- Call `clearCanvas(ctx, width, height)`.
- Compute two bounding boxes: left panel and right panel with padding between them.
  - `padding = 20`
  - `panelWidth = (width - padding * 3) / 2`
  - `panelHeight = height - padding * 2`
- Draw panel labels: structure name centered at top of each panel using theme font, bold.
- Look up source module: `structures[state.sourceKey].module`.
- Create source instance: `module.fromDyck(state.currentDyck)`.
- Call `module.draw(ctx, instance, { x, y, width: panelWidth, height: panelHeight - 30, theme, colors: CORRESPONDENCE_COLORS })` (subtract 30px for the label).
- Same for target on the right panel.
- Draw a thin vertical divider line between panels.

**Event handlers:**
- `#source-select` change: update `state.sourceKey`, call `render()`.
- `#target-select` change: update `state.targetKey`, call `render()`.
- `#n-select` change: update `state.n` (parseInt), call `updateDerivedState()`, call `render()`.
- `#btn-prev` click: decrement `state.instanceIndex` (wrap to last if at 0), call `updateDerivedState()`, call `render()`.
- `#btn-next` click: increment `state.instanceIndex` (wrap to 0 if at last), call `updateDerivedState()`, call `render()`.

**Initialization (DOMContentLoaded):**
1. Get canvas, call setupCanvas.
2. Read theme.
3. Populate dropdowns from registry.
4. Call updateDerivedState() to set initial Dyck words.
5. Call render() for initial display.
6. Attach all event handlers.
7. Resize handler: same 200ms debounce pattern, re-runs setupCanvas + render.

**Important implementation notes:**
- Store `ctx`, `width`, `height`, `theme` in module-scoped variables (not the state object) since they're rendering context, not application state.
- The playback control event handlers (btn-start, btn-step-back, btn-play-pause, btn-step-fwd, btn-end, speed-slider) should NOT be wired yet — leave them for Plan 03. But add a comment `// Playback controls wired in Plan 02-03`.
- Do NOT import or reference the animation engine — that doesn't exist yet.
