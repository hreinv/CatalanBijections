Create two new structure modules following the uniform interface pattern.

**staircase-polygon.js (STRC-05):**
- `fromDyck(dyckWord)`: Compute column heights from the Dyck word. Walk the word, tracking running height (prefix sum). At each position i, `currentHeight += dyckWord[i]`. Store `{ n, columns: Array<{x, height}>, dyckWord: [...dyckWord] }`. The columns represent the unit-square staircase under the Dyck path. Store a copy of the Dyck word for trivial round-trip (same pattern as parentheses.js storing the string directly).
- `toDyck(instance)`: Return `[...instance.dyckWord]` (trivial round-trip via stored copy).
- `draw(ctx, instance, opts)`: Render as a filled staircase region on a grid.
  - Use the "filled area under Dyck path" interpretation (simplest and most visually clear per research).
  - Draw light grid lines for context (2n x n grid, matching Dyck path scale).
  - For each column at position i with height h, fill a rectangle from baseline (y=0) to y=h at x-position i. Use semi-transparent fill (`colors[columnIndex % colors.length] + '40'`).
  - Draw the staircase outline (top boundary) with thick strokes. The outline traces a staircase pattern: for each column, draw horizontal line at that column's height, then vertical line to next column's height.
  - Draw the baseline at y=0 as a thin line.
  - The visual should clearly show a staircase shape, not just a Dyck path.

**rooted-plane-tree.js (STRC-07):**
- `fromDyck(dyckWord)`: DFS reconstruction. Start at root node `{ children: [] }`. Walk the Dyck word: +1 = create new child of current node, push current to stack, descend to child. -1 = pop stack, ascend to parent. Return the root node.
- `toDyck(instance)`: DFS serialization. Recursively visit each child: push +1 before descending, push -1 after returning. Return the accumulated word array.
- `draw(ctx, instance, opts)`: Render tree with width-accumulation layout to prevent collisions.
  - Implement a local `layoutPlaneTree(node, depth, xOffset)` function (NOT imported from another module):
    - Leaf nodes: `node.layoutX = xOffset + 0.5`, `node.layoutY = depth`, return width 1
    - Internal nodes: place children side by side using accumulated widths, center parent over first and last child x-coordinates
    - Return total subtree width
  - After layout, scale layoutX/layoutY to fit within bounding box with padding
  - Compute the total tree width and max depth to determine scaling factors
  - Draw edges from parent to children as straight lines (`theme.strokeColor`, `theme.strokeWidth || 3`)
  - Draw nodes as filled circles with correspondence colors, radius scaled to spacing
  - Draw the root node distinctly (slightly larger or with a border)
  - Handle degenerate cases: linear chain (depth 4, width 1) must spread vertically; star graph (depth 1, width 4) must spread horizontally. The width-accumulation algorithm naturally handles this.

CRITICAL for rooted plane trees (from Success Criteria #4): Verify no node collisions for all 14 instances at n=4 by ensuring every node gets a unique (layoutX, layoutY) pair.

Both modules: NO imports from other structure modules. Use the standard `{ x, y, width, height, theme, colors }` opts pattern.
