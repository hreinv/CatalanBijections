Create three new structure modules following the exact uniform interface pattern from Phase 2 (each exports `fromDyck`, `toDyck`, `draw`).

**mountain-range.js (STRC-03):**
- `fromDyck(dyckWord)`: Compute the same point array as dyck-path.js (step through Dyck word, accumulate x and y). Return `{ points: Array<{x, y}> }`.
- `toDyck(instance)`: Compute y-deltas between consecutive points (identical logic to dyck-path.js).
- `draw(ctx, instance, opts)`: Render as a filled mountain silhouette, NOT as a lattice grid path. Key visual differences from Dyck path:
  - NO grid lines (this is a silhouette, not a lattice)
  - Fill the area under the mountain profile with semi-transparent color (`colors[0] + '30'`)
  - Trace the path with `lineTo`, then close to baseline (`lineTo(endX, baseY)`, `lineTo(startX, baseY)`, `closePath()`, `fill()`)
  - Draw the mountain outline with thick strokes (`theme.strokeColor`, `theme.strokeWidth || 3`)
  - Draw a thin baseline at y=0
  - Mark peaks (local maxima where `points[i].y > points[i-1].y && points[i].y > points[i+1].y`) with colored dots using correspondence colors
  - Use same padding (20px) and scaling approach as dyck-path.js

**lattice-path.js (STRC-04):**
- `fromDyck(dyckWord)`: Map +1 to right step (R), -1 to up step (U). Compute points starting at (0,0): R increments x, U increments y. Return `{ n, steps: Array<'R'|'U'>, points: Array<{x, y}> }`.
- `toDyck(instance)`: Map steps back: 'R' -> +1, 'U' -> -1.
- `draw(ctx, instance, opts)`: Render on an n x n grid.
  - Draw light grid lines for the n x n grid
  - Draw the diagonal from (0,0) to (n,n) as a dashed colored line (using `ctx.setLineDash([5,5])`) -- this is the boundary the path must not cross above
  - IMPORTANT Canvas y-axis inversion: grid point (gx, gy) maps to canvas with y flipped so (0,0) is bottom-left and (n,n) is top-right
  - Draw the monotonic path with thick strokes
  - Color each step segment with correspondence colors (step i uses `colors[i % colors.length]`)
  - Draw vertex dots at grid intersections on the path

**ballot-sequence.js (STRC-08):**
- `fromDyck(dyckWord)`: Map +1 to 'A', -1 to 'B'. Compute running tallies. Return `{ votes: Array<'A'|'B'>, tallies: Array<{a, b}> }`.
- `toDyck(instance)`: Map votes back: 'A' -> +1, 'B' -> -1.
- `draw(ctx, instance, opts)`: Render as a row of labeled vote boxes with running tallies.
  - Render vote letters ("A" or "B") in a horizontal row, each colored with correspondence colors (similar to parentheses.js character rendering)
  - Below each vote, render the running tally as smaller text "A:X B:Y"
  - Use monospace font from theme (`theme.monoFont`)
  - Scale font size to fill bounding box width (same approach as parentheses.js)
  - Center horizontally and vertically within bounding box

All three modules: NO imports from other structure modules (keep independent). Use the same opts destructuring pattern `{ x, y, width, height, theme, colors }`.
