**non-crossing-partition.js (STRC-09):**

CRITICAL: The bijection convention needs careful implementation. Use the "matched pairs as elements, sibling pairs at same nesting depth form blocks" approach. Test all 14 instances at n=4 for round-trip correctness.

- `fromDyck(dyckWord)`: Convert Dyck word to a non-crossing partition of {1, ..., n}.
  1. Find all n matched pairs using a stack: walk the word, push position on +1, pop on -1. Number pairs 1..n by the order their opening +1 appears (left to right).
  2. Determine blocks: elements (pairs) that are "siblings" belong to the same block. Two pairs are siblings if they are at the same nesting depth within the same enclosing pair (or both at the top level with no enclosing pair).
  3. Implementation approach: maintain a stack of "current enclosing context." When encountering +1, the current element joins the block for the current depth/context. Track which block is active at each depth.
  4. Return `{ n, partition: Array<Array<number>> }` where each inner array is a block of 1-indexed elements, sorted by minimum element.

  IMPORTANT: If round-trips fail with the sibling-based approach, fall back to storing the Dyck word directly (like staircase-polygon.js) and reconstructing from it. Correctness is more important than algorithmic elegance.

- `toDyck(instance)`: Reconstruct Dyck word from partition.
  1. Sort blocks by their minimum element.
  2. Build a tree of blocks based on nesting structure: block B is a child of block A if B's elements are "between" consecutive elements of A.
  3. Serialize the block tree as a Dyck word: for each element in a block, write +1, recurse into child blocks between this element and the next, write -1.

  FALLBACK: If the toDyck algorithm is too complex or fragile, store the Dyck word in the instance (like staircase-polygon.js does) and return it directly. The round-trip test is the ultimate arbiter -- if it passes for all 14 instances at n=4, the implementation is correct regardless of approach.

- `draw(ctx, instance, opts)`: Render partition on a circle with arcs.
  - Place n points evenly around a circle (starting at top, -PI/2, going clockwise).
  - Use `Math.cos(angle)` and `Math.sin(angle)` for point positions (same approach as triangulation.js polygon vertices).
  - For each partition block with 2+ elements, draw arcs connecting all pairs of elements in the block as chords or quadratic curves toward the circle center. Color each block with a different correspondence color (`colors[blockIndex % colors.length]`), line width 3.
  - Draw filled circles at each point (radius 6).
  - Draw labels (1..n) outside the circle (offset slightly from the point along the radius).
  - For singleton blocks, just draw the point with no arcs.
  - Handle n=1 (single point, no arcs) and n=2 (two points, one possible arc) gracefully.

**stack-sortable-perm.js (STRC-11):**

- `fromDyck(dyckWord)`: Simulate Dyck-to-permutation. Walk the Dyck word: +1 = push next input value (1, 2, 3, ...) onto stack, -1 = pop stack top and append to output permutation. Return `{ perm: number[], n }`.

- `toDyck(instance)`: Simulate stack-sorting (Knuth's algorithm) to reconstruct the Dyck word.
  1. Initialize: empty stack, `nextOutput = 1`, `permIdx = 0`, `word = []`.
  2. Loop while `permIdx < n` OR stack is non-empty:
     - If stack top === nextOutput: pop, push -1 to word, increment nextOutput.
     - Else if permIdx < n: push perm[permIdx], push +1 to word, increment permIdx.
     - Else: pop, push -1 to word, increment nextOutput.
  CRITICAL: Use greedy popping -- ALWAYS check if the stack top matches nextOutput FIRST (before pushing). This ensures consecutive pops happen correctly. The condition check order matters: pop before push.

- `draw(ctx, instance, opts)`: Render the permutation as a row of labeled boxes.
  - Display the permutation values as a horizontal row of numbered boxes (similar to ballot-sequence.js layout).
  - Each box contains the permutation value (e.g., "2", "1", "3", "4").
  - Color each box with correspondence colors based on position index.
  - Use monospace font from theme. Scale font size to fill width.
  - Add a small label above or below: "Permutation: [2, 1, 3, 4]" or just render the boxes.
  - STRC-11 says "rendered with stack visualization" -- for static display, render the permutation prominently. The full stack visualization is better suited for Phase 5 bijection animation. For Phase 4, focus on clear permutation display with indication this is stack-sortable.

Both modules: NO imports from other structure modules. Standard opts pattern.
