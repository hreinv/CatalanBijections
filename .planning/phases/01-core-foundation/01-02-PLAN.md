**css/style.css:** Theme and layout:
- `:root` with CSS custom properties: `--bg-color: #FFFFFF`, `--stroke-color: #1A1A1A`, `--stroke-width: 3`, `--font-size: 18px`, `--font-family: 'Segoe UI', system-ui, sans-serif`, `--mono-font: 'Consolas', 'Courier New', monospace`, `--node-radius: 16`, `--padding: 24px`

- `#main-canvas` with width 100%, height 100%, display block
- No external font imports

**js/core/canvas-utils.js:** Two exported functions:
- `setupCanvas(canvas)`: Get devicePixelRatio (default 1), get boundingClientRect, set canvas.width/height to rect dimensions * dpr, set canvas.style.width/height to rect dimensions + 'px', get context with `{ alpha: false }`, apply `ctx.scale(dpr, dpr)`, return `{ ctx, width: rect.width, height: rect.height, dpr }`
- `clearCanvas(ctx, width, height)`: Fill entire rect with white (#FFFFFF) using fillRect (NOT clearRect, since alpha is false)

<task type="auto">
  <name>Task 2: Wire main.js entry point with canvas initialization and resize handling</name>
  <files>js/main.js</files>
  <action>
Create `js/main.js` as the application entry point:

1. Import `setupCanvas` and `clearCanvas` from `./core/canvas-utils.js`
2. Import `THEME_COLORS` from `./core/colors.js` (will be created by Plan 01; if not yet available, import will fail gracefully since module loading is deferred)
3. On DOMContentLoaded:
   - Get canvas element by id 'main-canvas'
   - Call setupCanvas(canvas) to get { ctx, width, height, dpr }
   - Call clearCanvas(ctx, width, height) to set white background
   - Read theme values from CSS custom properties via `getComputedStyle(document.documentElement)`: strokeColor, strokeWidth, fontSize, nodeRadius, fontFamily, monoFont
   - Store in a THEME object
   - Draw a verification pattern to prove canvas works: draw the text "Catalan Bijection Explorer" centered at top using theme font and stroke color, draw a horizontal line across the canvas at 1/3 height using theme stroke width, draw a small filled circle (radius from theme nodeRadius) at center of canvas. This verification drawing confirms HiDPI sharpness and theme application.
   - Log to console: `"Canvas initialized: ${width}x${height} @${dpr}x"`

4. Resize handler with 200ms debounce:
   - On window resize, clearTimeout existing timer, set new 200ms timeout
   - In timeout callback: re-run setupCanvas(canvas), clearCanvas, re-read theme (CSS values may change), redraw verification pattern
   - This prevents thrashing during drag-resize

Note: The import of colors.js is intentional even though Plan 01 creates it. Since both plans are Wave 1, the executor will handle ordering. If colors.js doesn't exist when this runs, the module will fail to load -- that's acceptable because Plan 01 creates it. The executor runs both plans and the final state will have both files.

IMPORTANT: Do NOT use global mutable state. Keep canvas state in a closure or module-scoped const. The app controller in Phase 2 will own application state.
  </action>
