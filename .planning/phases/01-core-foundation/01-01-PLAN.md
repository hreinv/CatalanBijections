  <name>Dyck Word Enumeration and Validation</name>
  <files>js/core/dyck.js, js/core/colors.js, tests/test-dyck.html</files>
  <behavior>
    Dyck word enumeration:
    - enumerate(1) -> [[1, -1]] (1 word)
    - enumerate(2) -> 2 words, each length 4
    - enumerate(3) -> 5 words, each length 6
    - enumerate(4) -> 14 words, each length 8
    - enumerate(0) -> throws RangeError
    - enumerate(5) -> throws RangeError

    Dyck word validation:
    - validate([1, -1]) -> true
    - validate([1, 1, -1, -1]) -> true
    - validate([-1, 1]) -> false (negative prefix sum)
    - validate([1, 1, -1]) -> false (non-zero final sum)
    - validate([]) -> true (empty word, sum=0)

    Every word returned by enumerate(n) must pass validate().
    Words use +1/-1 representation (maps to up/down steps, open/close parens).

    Color palette:
    - CORRESPONDENCE_COLORS: array of 8 hex color strings
    - THEME_COLORS: object with background, stroke, gridLine, highlight, text keys
    - All colors high-contrast on white background, projector-safe
  </behavior>
