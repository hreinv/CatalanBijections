**ballot-dyck.js (BIJC-08):**
- META: source='ballot-sequence', target='dyck-path', label='Ballot Sequences to Dyck Paths'
- Import ballot-sequence.js and dyck-path.js structure modules
- getSteps(dyckWord, n, reversed) produces 2n+2 steps:
  - Step 0: Show ballot sequence votes at default color (left), empty Dyck grid (right)
  - Steps 1..2n: For each vote i, highlight vote letter and draw corresponding Dyck path segment. Vote 'A' maps to up step (+1,+1), vote 'B' maps to down step (+1,-1). Description: "Vote {i+1}: '{A/B}' maps to {up/down} step"
  - Step 2n+1: Both fully colored with correspondence
- Drawing helpers for ballot side: render each vote letter (A or B) with three-zone coloring (processed=colored, active=pulsing glow, pending=dimmed at 0.25 alpha). Include the tally count below each vote showing cumulative A:B counts, dimming tallies the same way as votes.
- Dyck path side: reuse the exact drawDyckGrid/drawDyckSegments/drawDyckSegmentsComplete pattern from parens-dyck.js (copy the helper functions, do not import -- each bijection module is self-contained per Phase 3 decision 03-02).
- Reversed: swap sourceBox/targetBox per standard pattern, reverse steps array.
- Pulse: `0.5 + 0.5 * Math.sin(Date.now() * 0.008 * Math.PI)` for ~2Hz glow oscillation.
- Use ctx.save()/ctx.restore() around every element draw to prevent state leaks.

**dyck-mountain.js (BIJC-06):**
- META: source='dyck-path', target='mountain-range', label='Dyck Paths to Mountain Ranges'
- Import dyck-path.js and mountain-range.js structure modules
- getSteps(dyckWord, n, reversed) produces 2n+2 steps:
  - Step 0: Show Dyck path on grid (left), empty mountain baseline (right)
  - Steps 1..2n: For each step i, highlight Dyck segment i and draw corresponding mountain slope segment. Description: "Step {i+1}: {up/down} path step becomes {ascending/descending} mountain slope"
  - Step 2n+1: Both fully colored
- Drawing helpers for mountain side: render mountain profile building incrementally. For each segment i, draw a line from the current point to the next point (same x,y coordinates as Dyck path but rendered as a filled silhouette). Use beginPath/lineTo to build the mountain outline from (0,0) along the path, then close back to baseline with a fill at low alpha (0.15) for the area under the profile. Stroke the mountain outline at full opacity with correspondence colors per segment.
- Dyck path side: copy drawDyckGrid/drawDyckSegments/drawDyckSegmentsComplete helpers from parens-dyck.js.
- Color mountain segments by segment index (same as Dyck segments). Do NOT use the peak-only coloring from the static mountain-range draw() function (per Pitfall 2 in RESEARCH.md).
- Reversed: standard srcBox/tgtBox swap + steps.reverse().

Both modules should be approximately 200-350 lines following the structure of parens-dyck.js.
