Create the NCP-to-Triangulations bijection module following the established contract.

**ncp-triang.js (BIJC-07):**
- META: source='non-crossing-partition', target='triangulation', label='Non-crossing Partitions to Triangulations'
- Import non-crossing-partition.js, triangulation.js, and binary-tree.js structure modules
- getSteps(dyckWord, n, reversed) produces n+2 steps:
  - Step 0: Show NCP on circle with n labeled points (left panel), polygon (n+2)-gon outline only (right panel)
  - Steps 1..n: For each matched pair in the Dyck word (which determines both an NCP element and a triangle):
    - Highlight the corresponding element in the NCP with correspondence color
    - Reveal the corresponding triangle in the triangulation with same color
    - Description: "Element {i+1}: partition point maps to triangle ({v0}, {v1}, {v2})"
  - Step n+1: All NCP elements colored, all triangles filled with correspondence colors

**Correspondence computation (pre-computed in getSteps, not in drawFrame):**
1. Build the NCP from dyckWord via nonCrossingPartition.fromDyck(dyckWord) -- returns { n, blocks } where blocks is array of arrays (each block contains element indices)
2. Build the triangulation from dyckWord via triangulation.fromDyck(dyckWord) -- returns { triangles } where triangles is array of [v0, v1, v2] vertex triples
3. Build the binary tree from dyckWord via binaryTree.fromDyck(dyckWord) to get the implicit tree structure
4. The correspondence is: the i-th +1 in the Dyck word creates both NCP element i and binary tree node i (pre-order). Pre-order traversal of the binary tree determines triangle order (same as binary-triang.js). So NCP element i (by opening order in Dyck word) corresponds to triangle i (by pre-order tree traversal).
5. Build an explicit map: elementIndex -> triangleIndex. Walk the Dyck word, tracking opening positions. The i-th opening (0-indexed) corresponds to the i-th triangle collected by pre-order traversal of the binary tree.
6. Also build elementToBlock map so we know which block each element belongs to (for drawing arcs between elements of the same block).

**Drawing approach (left panel -- NCP):**
- Draw n points equally spaced on a circle (reuse circle layout from non-crossing-partition.js draw() -- compute points as `angle = -PI/2 + 2*PI*k/n` for k=0..n-1, center at box center)
- Label each point with its index (1-indexed)
- For blocks with multiple elements, draw arcs between consecutive block members using quadraticCurveTo bowed 30% toward center (same as non-crossing-partition.js)
- CRITICAL (Pitfall 5 from RESEARCH.md): Color NCP elements INDIVIDUALLY by element index (not by block). Each element i gets `colors[i % colors.length]` to match its corresponding triangle. This differs from the static NCP draw() which colors by block.
- Three-zone rendering on elements: processed=colored, active=pulsing glow, pending=dimmed
- Draw block arcs in the color of their most recently processed element, or dimmed if no element in the block is processed yet

**Drawing approach (right panel -- Triangulation):**
- Draw (n+2)-gon vertices equally spaced on a circle (reuse polygon layout from binary-triang.js -- `polygonParams` pattern)
- Draw polygon outline edges
- For each triangle i (in pre-order correspondence order):
  - Draw filled triangle with `colors[i % colors.length]` at alpha 0.15 (processed) or 0.25 (active with pulse) or skip (pending)
  - Draw triangle edges (diagonals) with full-opacity correspondence color
- Three-zone rendering on triangles: processed=filled+stroked, active=filled+stroked+glow, pending=not drawn

**CRITICAL (Pitfall 4 from RESEARCH.md):** Both orderings (NCP elements by opening position, triangles by pre-order) ARE the same ordering from the Dyck word. The i-th +1 creates element i and node i. Verify at n=2: element 1 same color as triangle 1, element 2 same color as triangle 2.

- Pulse: standard `0.5 + 0.5 * Math.sin(Date.now() * 0.008 * Math.PI)`
- ctx.save()/ctx.restore() around every element draw
- Pre-compute all circle points, polygon points, triangle vertices, and correspondence map in getSteps closure
- Reversed: standard srcBox/tgtBox swap + steps.reverse()

Module should be approximately 300-450 lines.
