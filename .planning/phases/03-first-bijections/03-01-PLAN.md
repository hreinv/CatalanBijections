  <name>Task 1: Create easing utility and bijection router skeleton</name>
  <files>js/core/easing.js, js/bijections/router.js</files>
  <action>
Create `js/core/easing.js` with two named exports:

1. `easeInOutCubic(t)` -- Robert Penner's cubic ease-in-out, simplified to normalized [0,1] input/output:
   ```
   return t < 0.5 ? 4 * t * t * t : 1 - Math.pow(-2 * t + 2, 3) / 2;
   ```
   Clamp input t to [0,1] before computing.

2. `lerp(a, b, t)` -- Linear interpolation: `return a + (b - a) * t;`

Create `js/bijections/router.js` with:

- A `registry` object (module-scoped, not exported).
- A `register(mod)` function that reads `mod.META.source` and `mod.META.target`, then registers the module under both `source|target` and `target|source` keys. The `target|source` entry stores `{ module: mod, reversed: true }`.
- An exported `getSteps(sourceKey, targetKey, dyckWord, n)` function that looks up `sourceKey|targetKey` in the registry. If found, calls `entry.module.getSteps(dyckWord, n, entry.reversed)` and returns the result. If not found, returns `null`.
- NO bijection imports yet -- the router starts empty. Bijection modules will be imported and registered in Plan 02 and Plan 03.
- Export `register` as a named export so bijection modules can self-register or main.js can register them after import.

Follow the zero-dependency ES module pattern from Phase 1-2. Use JSDoc comments on all exported functions.
  </action>

**js/main.js changes:**

1. Add imports at top:
   ```javascript
   import { easeInOutCubic } from './core/easing.js';
   import { getSteps as routerGetSteps } from './bijections/router.js';
   ```

2. Cache DOM reference in DOMContentLoaded:
   ```javascript
   dom.stepDescription = document.getElementById('step-description');
   ```

3. Create a helper function `loadBijectionSteps()` that:
   - Calls `routerGetSteps(state.sourceKey, state.targetKey, state.currentDyck, state.n)`
   - If result is non-null, sets `state.animation.steps = result` and `state.animation.currentStep = 0` and `state.animation.progress = 0.0`
   - If result is null, sets `state.animation.steps = []`
   - Calls `updateStepDescription()`

4. Create `updateStepDescription()` function:
   - If `state.animation.steps.length > 0` and `state.animation.steps[state.animation.currentStep]` exists, set `dom.stepDescription.textContent` to `"Step ${state.animation.currentStep + 1} of ${state.animation.steps.length}: ${step.description}"`
   - Otherwise set textContent to the default message `"Select two structures to see a bijection animation."`
   - If router returned null (no bijection registered), set textContent to the structure pair names + "No bijection available for this pair."

5. Modify `render()` to support dual-mode rendering:
   - After the existing guard `if (!state.currentDyck) return;`, compute layout constants (padding, labelHeight, panelWidth, panelHeight) -- these already exist in the current render function.
   - Check if `state.animation.steps.length > 0 && state.animation.steps[state.animation.currentStep]`:
     - **Animation mode:** Apply easing: `const easedProgress = easeInOutCubic(state.animation.progress);`. Call `step.drawFrame(ctx, easedProgress, { sourceBox, targetBox, theme, colors: CORRESPONDENCE_COLORS, currentStep: state.animation.currentStep, totalSteps: state.animation.steps.length })`. Then call `updateStepDescription()`. Still draw panel labels and divider (move label/divider drawing to a shared helper that runs in both modes).
     - **Static mode:** Execute the existing structure rendering code (no change to current behavior).

6. Call `loadBijectionSteps()` at the end of `onSourceChange`, `onTargetChange`, `onNChange`, `onPrev`, `onNext` (after resetAnimation and updateDerivedState). This way, whenever the user changes structures or instances, the steps are recomputed. resetAnimation clears steps first; loadBijectionSteps repopulates them if a bijection exists.

7. Also call `loadBijectionSteps()` once after initial `updateDerivedState()` + `render()` in DOMContentLoaded.

IMPORTANT: The easing is applied in render() to the raw progress value before passing to drawFrame. The animation engine stores raw linear progress. This prevents double-easing (per Pitfall 2 from research). Do NOT modify animation.js to apply easing.

IMPORTANT: Do NOT change the existing static rendering path. When steps are empty, the existing side-by-side structure rendering continues to work exactly as before.
