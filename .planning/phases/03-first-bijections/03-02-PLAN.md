  <action>
Create `js/bijections/parens-dyck.js` implementing the Parentheses-to-Dyck-Paths bijection.

**Exports:**
- `META = { source: 'parentheses', target: 'dyck-path', label: 'Parentheses to Dyck Paths' }`
- `getSteps(dyckWord, n, reversed = false)` returning an array of step objects

**Step structure (2n+2 steps for a word of length 2n):**

Step 0 (Introduction):
- description: `Start with the balanced parentheses: "${parenStr}"`
- drawFrame: Draw full parentheses string on left panel at full opacity (no color yet). Draw empty Dyck path grid on right panel (x-axis, y-axis, gridlines). No path segments yet.

Steps 1 through 2n (one per character):
- description: `Character ${i+1}: '${char}' maps to ${direction} step (${char === '(' ? '+1,+1' : '+1,-1'})`
- drawFrame implements three visual zones for elements (indexed 0 to 2n-1):
  - **Already processed (index < i):** Full opacity (globalAlpha=1.0), colored with CORRESPONDENCE_COLORS[index % colors.length], no glow. Draw parenthesis character on left with its color. Draw corresponding path segment on right with matching color.
  - **Active (index === i):** Full opacity. Colored with CORRESPONDENCE_COLORS[i % colors.length]. Pulsing glow via `ctx.shadowBlur = 8 + pulse * 12` where `pulse = 0.5 + 0.5 * Math.sin(Date.now() * 0.008 * Math.PI)`. `ctx.shadowColor` = the element's correspondence color. On the left panel, the active character pulses. On the right panel, the path segment animates in using the `progress` parameter (lerp from segment start point to end point, so at progress=0 the segment has zero length, at progress=1 it's fully drawn).
  - **Not yet processed (index > i):** Dimmed (globalAlpha=0.25). Draw in default stroke color (not correspondence color). Parenthesis characters on left are dimmed. No path segments on right (haven't been drawn yet).

Step 2n+1 (Completion):
- description: `Bijection complete: each parenthesis maps to one path step`
- drawFrame: All elements at full opacity with their correspondence colors. No glow. Both structures fully rendered and colored.

**Drawing implementation details:**

Left panel (parentheses):
- Compute character positions: evenly spaced horizontally within sourceBox, vertically centered.
- Use monospace font from theme (theme.monoFont).
- Font size: compute from box height to be readable (e.g., `Math.min(sourceBox.height * 0.3, 48)` px).
- Each character drawn with `ctx.fillText()` using its assigned color.

Right panel (Dyck path):
- Draw a grid: horizontal at y=0,1,...,n with light gridlines. Vertical at x=0,1,...,2n.
- Compute grid scaling to fit within targetBox.
- Path starts at (0, 0). Each +1 step goes to (x+1, y+1), each -1 step goes to (x+1, y-1).
- Draw path segments as thick lines (strokeWidth from theme or 3px).
- Each segment colored with CORRESPONDENCE_COLORS[segmentIndex % colors.length].
- Draw small circles at path vertices for clarity.

**Reversed direction:**
When `reversed === true`:
- Reverse the steps array.
- Swap source/target semantics in descriptions: describe reading path steps and writing parentheses.
- In drawFrame, the source is now Dyck path (left panel) and target is parentheses (right panel). Swap which structure draws where based on `opts.sourceBox` and `opts.targetBox` -- the opts from main.js already have sourceBox=left, targetBox=right based on the user's dropdown selection, so the bijection module must draw accordingly. When reversed, draw the Dyck path in sourceBox (left) and parentheses in targetBox (right).

**Canvas state management:**
- Use `ctx.save()` / `ctx.restore()` around each element draw to prevent state leaks.
- Always reset `ctx.globalAlpha = 1.0` and `ctx.shadowBlur = 0` after drawing glowing elements.
- Clear shadow state with `ctx.shadowColor = 'transparent'` after each glowing element.

**Anti-patterns to avoid (from research):**
- Do NOT apply easing inside drawFrame -- progress is already eased by the render function in main.js.
- Do NOT use hardcoded step counts -- derive from `dyckWord.length`.
- Do NOT modify the existing structure module draw() functions -- draw elements directly in drawFrame.
- ALWAYS use CORRESPONDENCE_COLORS for element coloring during animation, never the default theme stroke color.
