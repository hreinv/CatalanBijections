Create `js/bijections/binary-triang.js` implementing Binary Trees to Triangulations bijection.

**Exports:**
- `META = { source: 'binary-tree', target: 'triangulation', label: 'Binary Trees to Triangulations' }`
- `getSteps(dyckWord, n, reversed = false)` returning step array

**Core algorithm:**
Each internal node of the binary tree corresponds to exactly one triangle in the (n+2)-gon triangulation. Use pre-order traversal to collect (node, triangle) correspondence pairs.

Helper function `collectPairs(node, lo, hi)`:
- If node is null, return.
- Count left subtree size: `leftSize = countNodes(node.left)`.
- Compute apex vertex: `apex = lo + leftSize + 1`.
- Push pair: `{ nodeData, triangle: [lo, apex, hi], depth }`.
- Recurse: `collectPairs(node.left, lo, apex, depth+1)`.
- Recurse: `collectPairs(node.right, apex, hi, depth+1)`.
- Call initially as `collectPairs(tree, 0, n + 1, 0)`.

Helper function `countNodes(node)`:
- If null, return 0.
- Return `1 + countNodes(node.left) + countNodes(node.right)`.

**Step structure (n+2 steps):**

Step 0 (Introduction):
- description: `Show the binary tree with ${n} internal nodes and the (${n+2})-gon outline`
- drawFrame: Draw full binary tree on left panel (using the existing tree layout algorithm from binary-tree.js -- re-implement the layout logic locally or call the structure's draw with appropriate options). Draw the (n+2)-gon outline on right panel with numbered vertices (0 through n+1), but no diagonals yet. All elements at full opacity, default colors.

Steps 1 through n (one per node, in pre-order):
- description: `Node ${i+1}: internal node maps to triangle (${tri[0]}, ${tri[1]}, ${tri[2]})`
- drawFrame visual zones (same 3-zone pattern as parens-dyck):
  - **Already processed (j < i):** Full opacity, correspondence colored. Tree node filled with its color. Triangle on right filled with semi-transparent version of same color, diagonals drawn with same color.

  - **Not yet processed (j > i):** Tree node dimmed (globalAlpha=0.25), drawn in default color. No triangle drawn on right.

Step n+1 (Completion):
- description: `Bijection complete: each internal node maps to one triangle in the (${n+2})-gon`
- drawFrame: All nodes colored, all triangles filled with matching colors. Full correspondence visible.

**Create `js/bijections/parens-binary.js`:**

Implements Parentheses to Binary Trees bijection using the 1L0R recursive decomposition.

**Exports:**
- `META = { source: 'parentheses', target: 'binary-tree', label: 'Parentheses to Binary Trees' }`
- `getSteps(dyckWord, n, reversed = false)` returning step array

**Core algorithm:**
The 1L0R decomposition: in a Dyck word, find the first +1 and its matching -1 (first position where prefix sum returns to 0). The content between them becomes the left subtree, the content after becomes the right subtree. Recurse.

Helper function `decompose(word, startIdx, depth)`:
- If word is empty, return.
- Find the split: scan word accumulating prefix sum. The split index is where the sum first returns to 0.
- Record a decomposition step: `{ openIdx: startIdx, closeIdx: startIdx + splitIdx, leftContent: word.slice(1, splitIdx), rightContent: word.slice(splitIdx + 1), depth }`.
- Recurse on left content: `decompose(leftContent, startIdx + 1, depth + 1)`.
- Recurse on right content: `decompose(rightContent, startIdx + splitIdx + 1, depth + 1)`.
- Call initially as `decompose(dyckWord, 0, 0)`.

This produces one decomposition step per internal node (n steps total).

**Step structure (n+2 steps):**

Step 0 (Introduction):
- description: `Start with the balanced parentheses: "${parenStr}"`
- drawFrame: Draw full parentheses string on left panel. Draw empty tree space on right panel (just a light background or nothing).

Steps 1 through n (one per decomposition):
- description: `Decomposition ${i+1}: matched pair at positions ${openIdx+1},${closeIdx+1} creates node at depth ${depth}`
- drawFrame visual zones:
  - **Already processed (j < i):** Full opacity, correspondence colored. The matched parenthesis pair is colored on the left. The corresponding tree node is drawn and colored on the right, with edges to its parent.
  - **Active (j === i):** Full opacity, correspondence colored, pulsing glow. The matched pair brackets pulse on the left. On the right, the new tree node animates in (e.g., fades in using progress for opacity, or scales up). The edge connecting it to its parent also animates in.
  - **Not yet processed (j > i):** Characters dimmed on the left. No tree node on the right.

  On the left panel, additionally highlight the left-content substring with a subtle bracket/underline to show which portion becomes the left subtree, and similarly for the right-content substring. Use a lighter shade or underline mark. This visual cue helps the viewer understand the recursive decomposition.

Step n+1 (Completion):
- description: `Bijection complete: nested parentheses define the binary tree structure`
- drawFrame: All parentheses colored, full tree drawn with all nodes colored correspondingly.

**Drawing details:**

Left panel (parentheses):
- Same character layout as parens-dyck.js: evenly spaced horizontally, monospace font. Reuse the same drawing approach for consistency.
- For each decomposition step, the matched pair (openIdx, closeIdx) shares the correspondence color.

Right panel (binary tree):
- Re-implement the tree layout locally (in-order traversal x-positioning, depth-based y-positioning) within targetBox. This is the same layout algorithm from binary-tree.js.
- As nodes are revealed step by step, only draw the nodes and edges that have been revealed so far.
- Each node is a filled circle with its correspondence color.
- Edges drawn in default stroke color.

**Important: Node-to-pair mapping.**
The decomposition order IS the pre-order traversal of the resulting tree. The first decomposition creates the root. The second creates the root's left child (or right if left subtree is empty). This means correspondence color index 0 maps to the root node and the outermost matched pair. Follow this indexing for consistent coloring.

**Reversed direction:**
When reversed, swap panel assignments and reverse step array. Describe it as reading tree structure to produce parentheses.
